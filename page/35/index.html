Binary Search Tree Iterator

Implement an iterator over a binary search tree (BST). Your iterator will be initialized with the root node of a BST.

Calling next() will return the next smallest number in the BST.

Note: next() and hasNext() should run in average O(1) time and uses O(h) memory, where h is the height of the tree.

Method

next操作相当于取出BST的最小节点并将其从树中删除。一直向左子结点遍历知道其左子结点为null时，即为这个树的左子最小节点，获得该节点值。另外需要将这个节点从BST中删除，还需要保持BST的性质。

若该节点没有子节点，则直接删除。
若该节点有右子节点，其父节点的左指针指向该节点的右子节点。
如果该节点为根节点，需要将根节点重新设置为根的右子节点。

遍历完了树就没了，破坏性太大。。然后时间复杂度是O(h)的，无法达到O(1)。上面的方法一般是用于BST删除最小节点时用的。

Implement Queue using Stacks

mplement the following operations of a queue using stacks.

push(x) – Push element x to the back of queue.
pop() – Removes the element from in front of queue.
peek() – Get the front element.
empty() – Return whether the queue is empty.

Notes:
You must use only standard operations of a stack – which means only push to top, peek/pop from top, size, and is empty operations are valid.
Depending on your language, stack may not be supported natively. You may simulate a stack by using a list or deque (double-ended queue), as long as you use only standard operations of a stack.
You may assume that all operations are valid (for example, no pop or peek operations will be called on an empty queue).

Method

使用两个栈，一个inStack，一个outStack。push的元素都放入inStack，需要pop或peek时，判断outStack是否为空，若为空则pop所有inStack中的元素并push到outStack中，然后pop或peek outStack的栈顶元素，若不为空，则直接pop或peek outStack的栈顶元素。

第二种方法是每次push时，使用一个辅助数组或辅助栈保存当前栈pop出的所有元素，当前栈为空，然后放入push的元素，再将辅助数组或辅助栈保存的元素按照正确顺序放回当前栈中。

Min Stack

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

push(x) – Push element x onto stack.
pop() – Removes the element on top of the stack.
top() – Get the top element.
getMin() – Retrieve the minimum element in the stack.

Method

关键是getMin操作，如果当前pop的是最小值，就需要遍历整个stack去搜索最小值，则pop的复杂度可能为O(n)。

增加minStack用于维护最小值，如果push的值小余minStack栈顶的值或minStack为空，则添加到minStack。

如果一个值m被添加到minStack，说明m是当前push的所有值中最小的，后面又push了很多值，任何比m大的值不会被加入到minStack中，直到出现第一个比m小的值n，会被加入到minStack中。不断pop出，minStack和record中的n是同时被pop出的，这是minStack栈顶为m，一直到minStack和record中的n同时被pop出的，恢复到m被push之前的状态。也就是minStack栈顶一直保持record中的元素的最小值。

另外在实现中遇到java一个需要很隐蔽但需要注意的问题。

Return false; 原因在将元素放入Stack，java会自动将整数装箱(boxing)，区间-128-127之外的整数自动装箱时会创建一个新的Integer对象，在判断表达式中两个都是Integer类型，因此不会拆箱也直接比较，默认==操作是比较两个对象的地址。

Return true; 原因是区间在-128-127中的元素在自动装箱时会指向同一个Integer对象。

Return true. 在判断式中进行了自动拆箱。

Return false，因为没有使用自动装箱。
