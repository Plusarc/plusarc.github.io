Ensure that numbers within the set are sorted in ascending order.

Method

DFS搜索解空间。把问题视为对于k个位置，递增填入1-9范围内的数字，使得sum为n。从第一个个位置开始，选择1或2,3…n-k，进行下一步深度优先搜索，同时记录当前所处的位置，目前已经使用的最大值(下一步搜索的值必须大于目前已经使用的最大值), 目前使用的所有数值，。

如果某一步递归时sum已经为n且使用了k个数字，则为一组可行解，添加到result中。
否则如果 当前位置已超过k，或者上一轮已经使用最大数字9，或者sum>n或sum=n && record.size!=k 则搜索失败，回溯。

另外一点可以避免不必要搜索是的是，剩余可选的数字必须大于等于剩余需要添加的数字数，max<=10-(k-pos), 也就是如果当前选择数字8后，后面可选择的就只有9了，但如果还有2两个数字要选，则这个搜索肯定是失败的，因为没有k个number。也就是当前不能选择8，cut掉所有选择8之后的搜索。

Find Minimum in Rotated Sorted Array

Suppose a sorted array is rotated at some pivot unknown to you beforehand.

(i.e., 0 1 2 4 5 6 7 might become 4 5 6 7 0 1 2).

Find the minimum element.

You may assume no duplicate exists in the array.

Method

二分法搜索的变化形式。关键是边界case的处理。

得到了accept solution但觉得边界处理不够系统和简洁，也不太好解释。

找到了比较简洁和系统的思路和solution，参考：
Find Minimum in Rotated Sorted Array I, II
Search in Rotated Sorted Array I

根据A[mid]和A[end]来判断右半数组是否sorted：
原数组：0 1 2 4 5 6 7
情况1： 6 7 0 1 2 4 5
情况2： 2 4 5 6 7 0 1

cases:
(1) A[mid] < A[end] ： A[mid : end] sorted => min不在A[mid+1 : end]中, 搜索A[start : mid]
(2) A[mid] > A[end] ： A[start : mid] sorted且又因为该情况下A[end]<A[start] => min不在A[start : mid]中, 搜索A[mid+1 : end]
(3) base case：
a. start = end，必然A[start]为min，为搜寻结束条件。
b. start + 1 = end，此时A[mid] = A[start]，而min = min(A[mid], A[end])。而这个条件可以合并到(1)和(2)中。

Better-solution

public class Solution {
public int findMin(int[] nums) {
int left = 0;
int right = nums.length-1;

    while(left<right) {
        int mid = (left + right) / 2;
        if(nums[mid]<nums[right]) right = mid;
        else left = mid + 1;
    }
    return nums[left];
}

}

House Robber

You are a professional robber planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security system connected and it will automatically contact the police if two adjacent houses were broken into on the same night.

Given a list of non-negative integers representing the amount of money of each house, determine the maximum amount of money you can rob tonight without alerting the police.

Method

其实就是对于一个数组，不能去任意相邻的两个值，求能取得的最大的sum。

Method 1

直接想到的是递归搜索解空间树，也就是对于当前的元素，选择加入或不加入，如果加入，则算入sum，然后从下下个开始继续递归。如果不加入，则sum不变，从下个元素开始继续递归。

得到这两种选择的较大值，返回结果。

要注意到的是，计算过程中其实有很多重复的递归计算。

Method 2

动态规划方法，设max[n]为从0到n的子数组能取得的最大的sum，注意到对于数组0到n能取到的最大的max[n]，为max[n-1]与max[n-2]+num[n]中的较大值，因为最大值必定为对n元素取或者不取结果的一种，如果取这其最大sum为max[n-2]+num[n]，如果不取，daon的最大sum为max[n-1]。这就是一个动态规划的问题了，也就是状态转移方程为：

Minimum Size Subarray Sum

Given an array of n positive integers and a positive integer s, find the minimal length of a subarray of which the sum ≥ s. If there isn’t one, return 0 instead.

For example, given the array [2,3,1,2,4,3] and s = 7,
the subarray [4,3] has the minimal length under the problem constraint.

Method

Method 1

使用滑动窗口，保证窗口内的元素之和大于target，并且尽量减少其长度，同时记录一个最小窗口长度值。具体是设置两个指针start和end记录窗口的开始和结束，右移end直到窗口之和大于等于target，右移start指针，直到start+1到end子窗口之和小余target，记录窗口大小，如果小余当前记录最小窗口，则更新最小窗口为目前窗口。继续右移end，这样重复操作直到扫描完数组。

Method 2

建立一个数组sum, 其中sum[i]记录从0到i-1的所有元素之和(这个数组是有序的，因为条件是原数组为非负数组)，如果sum[i]>=target，就2分查找比sum[i]-target+1小且是最大的数字，得到这个以i结尾大于target的子数组中尽量小的一个。比较其长度与当前记录最小长度并更新。

本质上与滑动窗口是相同的，都是找到每个位置结尾的大于target且尽量短的数组，然后取这些子数组中长度最小的一个。
