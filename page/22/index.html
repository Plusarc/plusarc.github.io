Binary Tree Zigzag Level Order Traversal

Given a binary tree, return the zigzag level order traversal of its nodes’ values. (ie, from left to right, then right to left for the next level and alternate between).

For example:
Given binary tree {3,9,20,#,#,15,7},

return its zigzag level order traversal as:

Method

跟树的层次遍历一样，仍使用前序遍历实现。采用LinkedList记录每一层的记录，如果是偶数层(0，2，4…)，则直接add记录，如果是奇数层(1,3,5…)，则使用addFirst添加记录(LinkedList长处)。

Sort Colors

Given an array with n objects colored red, white or blue, sort them so that objects of the same color are adjacent, with the colors in the order red, white and blue.

Here, we will use the integers 0, 1, and 2 to represent the color red, white, and blue respectively.

Note:
You are not suppose to use the library’s sort function for this problem.

Method

最直接的方法是对数组进行排序，时间复杂度为O(nlong(n))。

但是这里的情况是只有3种值，使用quick sort的思想，分两次将数组partition成有序。

首先将2的值全部移到数组的最右端，方法跟quick sort的partition一样，设置left，right指针，left不断右移直到指向一个等于2的值，right不断左移直到指向一个小于2的值(这里也就是0，1)，交换left，right的值。

不断重复直到left>right，这时right指向从右边开始的第一个非2的值，也就是right, length-1都是2，0-right为0，1组成。

设置left=0, right不变，重复上面的排序过程，不过这次目的是将1的元素移到0-right子数组的最右端。

操作完成后，整体数组有序，最左边为0,中间为1,右边2。（如果没有0,1,2的部分元素也是一样）。

Gray Code

The gray code is a binary numeral system where two successive values differ in only one bit.

Given a non-negative integer n representing the total number of bits in the code, print the sequence of gray code. A gray code sequence must begin with 0.

For example, given n = 2, return [0,1,3,2]. Its gray code sequence is:

Note:
For a given n, a gray code sequence is not uniquely defined.

For example, [0,2,3,1] is also a valid gray code sequence according to the above definition.

For now, the judge is able to judge based on one instance of gray code sequence. Sorry about that.

Method

本质上可以用DFS的思想解决，每次选择与当前数字只差1位的数字。每次改变当前数字一位，判断生成的这个数字是否已经访问过，如果没有访问过，则将其加入结果集中，同时加入记录已访问数字的HashSet中，以这个数为开始以同样的操作生成只只改变一位的下一个数字，知道生成所有的数字序列。

不断循环上面的操作知道生成2^n个不同数字列，也就是一组Gray Code.

Course Schedule

There are a total of n courses you have to take, labeled from 0 to n - 1.

Some courses may have prerequisites, for example to take course 0 you have to first take course 1, which is expressed as a pair: [0,1]

Given the total number of courses and a list of prerequisite pairs, is it possible for you to finish all courses?

For example:

2, [[1,0]]
There are a total of 2 courses to take. To take course 1 you should have finished course 0. So it is possible.

2, [[1,0],[0,1]]
There are a total of 2 courses to take. To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.

Note:
The input prerequisites is a graph represented by a list of edges, not adjacency matrices. Read more about how a graph is represented.

Method

课程的组织可以表示为一个有向图，问题本质是检测一个有向图中是否有环，如果一个图有环，则一定存在冲突，也就是环中的课程一定无法参加。对图进行拓扑排序，如果可以进行完整的拓扑排序(输出排序节点等于所有节点数)，则说明是一个有向无环图，否则说明有环。

这里采用基于队列的拓扑排序：

  1. 遍历所有边，统计所有节点的入度(就是修课前需要完成的课程数)，和指向的节点(用HashMap存储)。
  2. 将所有入度为0的节点加入队列。
  3. 在队列非空的情况下，取出队列的一个节点，将count(拓扑序列的长度)加1，将所有这个节点指向的节点的入度-1，如果每个被指向的节点入度变为0，则将其加入队列。
  4. 循环第3步直到队列为空，如果count==numCourses，则成功得到一个完整的拓扑序列，也就是所有课程都可以修完，如果count<numCourse，则存在环，部分课程无法参加。

H-Index

發表於 2015-09-04   |   分類於 leetcode   |  

H-Index

Given an array of citations (each citation is a non-negative integer) of a researcher, write a function to compute the researcher’s h-index.

According to the definition of h-index on Wikipedia: “A scientist has index h if h of his/her N papers have at least h citations each, and the other N − h papers have no more than h citations each.”

For example, given citations = [3, 0, 6, 1, 5], which means the researcher has 5 papers in total and each of them had received 3, 0, 6, 1, 5 citations respectively. Since the researcher has 3 papers with at least 3 citations each and the remaining two with no more than 3 citations each, his h-index is 3.

Note: If there are several possible values for h, the maximum one is taken as the h-index.

Method

理解了题目意思就很容易，虽然有点绕。

注意H-Index的两个条件：

  1. h of his/her N papers have at least h citations
  2. the other N − h papers have no more than h citations each.

H的选择是0-N。

本质上，H-index要满足的条件是citation大于H的文章数大于等于H。也就是，假设对于H的选择范围0-N设立一个数组cum[0-N]，cum[i]表示记录citation大于等于i的文章数。i为H-Index的条件为cum[i+1]<(i+1) && cum[i]>=i。这里需要注意=的情况，也就是citation=i的文章的数量，这些文章可以被分配到have at least h citations， 也可以被分配到have no more than h citations each. 也就是会形成一个可行的范围。

最简单的方法是依次选择每一个H，然后遍历数组计算出大于N的文章数，小于N的文章数，以及等于N的文章数，基于这些数量判断H是否是H-index。时间复杂度为O(N^2).

优化方法：建立一个数组count[]，count[i]表示citation_num==i 的文章数(最后一个元素是特殊的，count[count.length-1]表示citation_num>=N的文章数), 遍历citation数组依次，统计得到count[]数组。从大到小，每次累计前一个count[i+1]，也就是count[i]=count[i] + count[i+1]，累计之后count[i]也就是前面所用的cum[i]。时间复杂度为O(N).
