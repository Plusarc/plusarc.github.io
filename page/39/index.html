Combination Sum

Given a set of candidate numbers (C) and a target number (T), find all unique combinations in C where the candidate numbers sums to T.

The same repeated number may be chosen from C unlimited number of times.

Note:
All numbers (including target) will be positive integers.
Elements in a combination (a1, a2, … , ak) must be in non-descending order. (ie, a1 ≤ a2 ≤ … ≤ ak).
The solution set must not contain duplicate combinations.
For example, given candidate set 2,3,6,7 and target 7,
A solution set is:
[7]
[2, 2, 3]

Method

回溯法。先对数组进行排序。

对于递归中当前位置数组的数，可以选择不考虑这个数，跳到下一个数，或者选择这个数，留在当前位置，因为不限定一个数字的使用次数。如果sum已经大于target或者pos已经超出数组的范围，则搜索失败，返回。如果sum=target，则找到一个可行解，将其加入结果集(注意要复制该list，而不是直接添加引用)，返回。

Happy Number

Write an algorithm to determine if a number is “happy”.

A happy number is a number defined by the following process: Starting with any positive integer, replace the number by the sum of the squares of its digits, and repeat the process until the number equals 1 (where it will stay), or it loops endlessly in a cycle which does not include 1. Those numbers for which this process ends in 1 are happy numbers.

Example: 19 is a happy number

Method

注意题中所提到的，如果一个数是Happy number则最终会变为1，否则会loops endlessly in a cycle，也就是会重复一定的数字序列。

设置HashSet记录已经出现过的数字，如果变为1，则为Happy number。如果出现重复的数字，则不为Happy number。按照题中条件，任意一个数都是这两种情况的一种。

Unique Paths

A robot is located at the top-left corner of a m x n grid (marked ‘Start’ in the diagram below).

The robot can only move either down or right at any point in time. The robot is trying to reach the bottom-right corner of the grid (marked ‘Finish’ in the diagram below).

How many possible unique paths are there?

Method

Naive method

暴力搜索法，从开始位置向下或向右移动，到达终点则为一条路径，到达棋盘底端或右端且不是终点，则终止搜索返回, 可以使用递归实现.

DP method

首先要认识到这个问题是可以拆分为子问题的，目标是求得从(1,1)到(m,n)的路径，并每次只能向下或向右移动。假设p(x,y)是从(1,1)到(x,y)的路径数，则p(m,n) = p(m-1,n) + p(m,n-1), 因为到达(m,n)前一个点必为(m-1,n)或(m,n-1)。

所以问题可以写为：

p(x, 1) = 1, x>=1 && x<=m
p(1, y) = 1, y>=1 && y<=n
p(x, y) = p(x-1, y) + p(x, y-1), x>=2 && x<=m, y>=2 && y<=n

下面代码所有用于指示位置的下标都减1.

Follow-up

第二遍时发现，我们可以进一步降低空间复杂度。由于每一步只依赖于其上面和左边两个，所以我们只需要一行空间。

p[i] = p[i-1] + p[i]

Largest Number

Given a list of non negative integers, arrange them such that they form the largest number.

For example, given [3, 30, 34, 5, 9], the largest formed number is 9534330.

Note: The result may be very large, so you need to return a string instead of an integer.

Method

策略是总是让从数字位上较大的且较短的数字放在前面。

从字符串会比较好理解。如9, 91,，正确的顺序为9``91，而不是91``9，本质上是满足组合后的字典排序，991的字典排序在919之前。也就是要比较两个数哪个排在前面时，分别生成其两种组合的字符串，比较其字典排序确定更大的组合方式。

把原来int数组换为String数组，自定义Comprator对String进行排序，然后从前到后组合得到结果。
