Invert Kth Smallest Element in a BST

發表於 2015-08-02   |   分類於 leetcode   |  

Kth Smallest Element in a BST

Given a binary search tree, write a function kthSmallest to find the kth smallest element in it.

Note:
You may assume k is always valid, 1 ≤ k ≤ BST’s total elements.

Follow up:
What if the BST is modified (insert/delete operations) often and you need to find the kth smallest frequently? How would you optimize the kthSmallest routine?

Method

递归解决。一个节点在BST的位置为 左子树节点数+1。
递归搜索左子树，如果K在左子树中，则直接返回搜索左子树返回的结果。
否则根据搜索左子树的累计节点数，判断是当前节点是否为K，若是，则返回结果。
若不是则递归搜索右子树，搜索的位置为 K-左子树累计节点数-1.
若在右子树中，则直接返回搜索右子树返回的结果。
若也不在右子树中，则累计当前树的所有节点数，返回上一层。

Combination Sum II

Given a collection of candidate numbers (C) and a target number (T), find all unique combinations in C where the candidate numbers sums to T.

Each number in C may only be used once in the combination.

Note:
All numbers (including target) will be positive integers.
Elements in a combination (a1, a2, … , ak) must be in non-descending order. (ie, a1 ≤ a2 ≤ … ≤ ak).
The solution set must not contain duplicate combinations.
For example, given candidate set 10,1,2,7,6,1,5 and target 8, A solution set is:[1, 7]`[1, 2, 5]
[2, 6]
[1, 1, 6]

Method

跟Combination Sum类似，不过还好处理一些，因为每个数字只能用一次。

仍然是回溯法。先对数组进行排序。

对于递归中当前位置数组的数，可以选择不考虑这个数，跳到下一个数，或者选择这个数，跳到下一个位置，因为每个数字只能用一次。如果sum已经大于target或者pos已经超出数组的范围，则搜索失败，返回。如果sum=target，则找到一个可行解，将其加入结果集(注意要复制该list，而不是直接添加引用)，返回。

Unique Paths

A robot is located at the top-left corner of a m x n grid (marked ‘Start’ in the diagram below).

The robot can only move either down or right at any point in time. The robot is trying to reach the bottom-right corner of the grid (marked ‘Finish’ in the diagram below).

How many possible unique paths are there?

Method

Naive method

暴力搜索法，从开始位置向下或向右移动，到达终点则为一条路径，到达棋盘底端或右端且不是终点，则终止搜索返回, 可以使用递归实现.

DP method

首先要认识到这个问题是可以拆分为子问题的，目标是求得从(1,1)到(m,n)的路径，并每次只能向下或向右移动。假设p(x,y)是从(1,1)到(x,y)的路径数，则p(m,n) = p(m-1,n) + p(m,n-1), 因为到达(m,n)前一个点必为(m-1,n)或(m,n-1)。

所以问题可以写为：

p(x, 1) = 1, x>=1 && x<=m
p(1, y) = 1, y>=1 && y<=n
p(x, y) = p(x-1, y) + p(x, y-1), x>=2 && x<=m, y>=2 && y<=n

下面代码所有用于指示位置的下标都减1.

Follow-up

第二遍时发现，我们可以进一步降低空间复杂度。由于每一步只依赖于其上面和左边两个，所以我们只需要一行空间。

p[i] = p[i-1] + p[i]

Largest Number

Given a list of non negative integers, arrange them such that they form the largest number.

For example, given [3, 30, 34, 5, 9], the largest formed number is 9534330.

Note: The result may be very large, so you need to return a string instead of an integer.

Method

策略是总是让从数字位上较大的且较短的数字放在前面。

从字符串会比较好理解。如9, 91,，正确的顺序为9``91，而不是91``9，本质上是满足组合后的字典排序，991的字典排序在919之前。也就是要比较两个数哪个排在前面时，分别生成其两种组合的字符串，比较其字典排序确定更大的组合方式。

把原来int数组换为String数组，自定义Comprator对String进行排序，然后从前到后组合得到结果。
