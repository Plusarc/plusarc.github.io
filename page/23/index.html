Gas Station

There are N gas stations along a circular route, where the amount of gas at station i is gas[i].

You have a car with an unlimited gas tank and it costs cost[i] of gas to travel from station i to its next station (i+1). You begin the journey with an empty tank at one of the gas stations.

Return the starting gas station’s index if you can travel around the circuit once, otherwise return -1.

Note:
The solution is guaranteed to be unique.

Method

理解题目意思很重要，给定围成圈的一些加油站，cost[i]表示从加油站i到加油站i+1所需要花费的油，cost[i]表示从加油站i所存储的油。问从哪个加油站出发可以回到加油站(方向为向右，cost[i]提供的信息)，有唯一一个解或没有解。

例如:

若从第一个加油站出发，则会卡在第4个加油站，无法到达第5个加油站。
若从第二个加油站出发，同样会卡在第4个加油站。

若从第5个加油站出发，则能绕一圈回到第五个加油站，也就是解。

关键的一点是要分析出，从一个加油站i出发(若其不是解，也就是会卡在某个加油站j)，则从i到j的加油站(包括i，j)，都不可能是可行解。

原因:

  1. 假设k是i到j中间的一个加油站(包括i, j)，已k为起点，假设其能抵达j。
  2. 如果需要从加油站j抵达下一个加油站j+1，其条件是，从起点k到达j累计下来的gas（记为cum(k,j), cum(k,j)>=0,否则无法从j-1抵达j），加上加油站j的gas[j]，必须大于从j到j+1所需要的cost[j]。也就是需要cum(k, j)+gas[j]>=cost[j]
  3. 目前知道的是从i出发会卡在j，也就是 cum(i, j)+gas[i]<cost[j]，而cum(i, j) = cum(i, k) + cum(k, j)，而cum(i, k)>=0(否则无法从k-1抵达k)，也就是我们可以得到cum(i, j)>=cum(k, j), 又cum(i, j)+gas[i]<cost[j]，所以cum(k, j)+gas[i]<cost[j]，也就是无法从j抵达j+1.

结论：
从一个加油站i出发(若其不是解，也就是会卡在某个加油站j)，则从i到j的加油站(包括i，j)，都不可能是可行解。

也就是我们无需计算中间的这些加油站的可行性，直接跳到计算j+1加油站的可行性，当从某个加油站出发，向前走了num-1步(num为加油站个数)，并且下一步也可行，则找到了可行解。

如果判断所有加油站都不可行，则无解。

注意处理circle的情况，用取余%实现，注意边界情况。

虽然代码有两个循环，但是两个循环是同时推进的，时间复杂度为O(N), Greedy策略.

Word Ladder

Given two words (beginWord and endWord), and a dictionary, find the length of shortest transformation sequence from beginWord to endWord, such that:

Only one letter can be changed at a time
Each intermediate word must exist in the dictionary
For example,

Given:
start = "hit"
end = "cog"
dict = ["hot","dot","dog","lot","log"]
As one shortest transformation is "hit" -> "hot" -> "dot" -> "dog" -> "cog",
return its length 5.

Note:
Return 0 if there is no such transformation sequence.
All words have the same length.
All words contain only lowercase alphabetic characters.

Method

Method 1

BFS问题，求最短路径。判断set中可以抵达endWord的词，并判断其是否可以由beginWord抵达，若是，则直接返回距离，若不能，加入队列中。弹出队首，判断set中可以抵达弹出的队首的词，同样，判断其是否可以由beginWord抵达，若是，则直接返回距离，若不能，加入队列中…直到找到可以有beginWord抵达第队列中的词或队列为空。

对于判断set中能抵达当前词的词，有两种方式，一种是对于当前词，依次对每个字母分别进行改变，判断改变后的词是否在set中，复杂度为O(25*l), 每个字母有25种选择，l为word的长度，是method 1使用的方法。另一种是method 2中的方法，考虑到dict的大小，第一种方法比较高效。

另外同时注意记录当前当前处理的词的距离，以及每次移除已经遍历过的节点(添加到queue中的节点)。

Method 2

思路相同，只是对于判断set中能抵达当前词的词，逐一扫描set中所有的词，复杂度为O(s*l), s为dict的大小，l为word的长度。当dict较大时，效率明显低于method1，会超时。

Simplify Path

Given an absolute path for a file (Unix-style), simplify it.

For example,
path = "/home/", => "/home"
path = "/a/./b/../../c/", => "/c"

Method

解析路径，分离出每个子元素，并用栈记录路径。

子元素为.，则无需操作，进行解析下一个子元素。
子元素..，则在栈不为空的情况下，返回上一级，也就是弹出栈顶元素。
子元素为其他（包括 ... 的情况），则压入栈。

另外注意处理一些特殊情况，例如 //, /..., /a//b

也就是一些末尾不是/的情况，或者/连续出现的情况。
