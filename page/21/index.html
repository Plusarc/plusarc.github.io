Populating Next Right Pointers in Each Node

Given a binary tree

struct TreeLinkNode {
  TreeLinkNode *left;
  TreeLinkNode *right;
  TreeLinkNode *next;
}

Populate each next pointer to point to its next right node. If there is no next right node, the next pointer should be set to NULL.

Initially, all next pointers are set to NULL.

Note:
You may only use constant extra space.
You may assume that it is a perfect binary tree (ie, all leaves are at the same level, and every parent has two children).

For example,
Given the following perfect binary tree,

After calling your function, the tree should look like:

Method

Method 1

由于条件是一棵perfect，每个节点都有确定的序号，并且可以确定是否是否右驱节点，如果有，其右驱节点序号为当前节点序号+1。

这样最直接的方法是建立一个数组，遍历树一遍，将节点的引用存入数组对应的位置(序号位置)。然后再遍历树一遍，设置next的值。

需要O(n)的空间。

后面又想了下面的方法2，无需额外空间，实现也更简洁，所以就没有实现Method 1，这里只给出了Method 2的实现.

Method 2

对于一个节点如果左右子节点非空，则设置root.left.next = root.right.

另外如果其next!=null，则设置root.right.next = root.next.left，也就是其右子节点的next指向其next节点的左子结点。

递归地对左右子树进行同样操作。

Gray Code

The gray code is a binary numeral system where two successive values differ in only one bit.

Given a non-negative integer n representing the total number of bits in the code, print the sequence of gray code. A gray code sequence must begin with 0.

For example, given n = 2, return [0,1,3,2]. Its gray code sequence is:

Note:
For a given n, a gray code sequence is not uniquely defined.

For example, [0,2,3,1] is also a valid gray code sequence according to the above definition.

For now, the judge is able to judge based on one instance of gray code sequence. Sorry about that.

Method

本质上可以用DFS的思想解决，每次选择与当前数字只差1位的数字。每次改变当前数字一位，判断生成的这个数字是否已经访问过，如果没有访问过，则将其加入结果集中，同时加入记录已访问数字的HashSet中，以这个数为开始以同样的操作生成只只改变一位的下一个数字，知道生成所有的数字序列。

不断循环上面的操作知道生成2^n个不同数字列，也就是一组Gray Code.

Course Schedule

There are a total of n courses you have to take, labeled from 0 to n - 1.

Some courses may have prerequisites, for example to take course 0 you have to first take course 1, which is expressed as a pair: [0,1]

Given the total number of courses and a list of prerequisite pairs, is it possible for you to finish all courses?

For example:

2, [[1,0]]
There are a total of 2 courses to take. To take course 1 you should have finished course 0. So it is possible.

2, [[1,0],[0,1]]
There are a total of 2 courses to take. To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.

Note:
The input prerequisites is a graph represented by a list of edges, not adjacency matrices. Read more about how a graph is represented.

Method

课程的组织可以表示为一个有向图，问题本质是检测一个有向图中是否有环，如果一个图有环，则一定存在冲突，也就是环中的课程一定无法参加。对图进行拓扑排序，如果可以进行完整的拓扑排序(输出排序节点等于所有节点数)，则说明是一个有向无环图，否则说明有环。

这里采用基于队列的拓扑排序：

  1. 遍历所有边，统计所有节点的入度(就是修课前需要完成的课程数)，和指向的节点(用HashMap存储)。
  2. 将所有入度为0的节点加入队列。
  3. 在队列非空的情况下，取出队列的一个节点，将count(拓扑序列的长度)加1，将所有这个节点指向的节点的入度-1，如果每个被指向的节点入度变为0，则将其加入队列。
  4. 循环第3步直到队列为空，如果count==numCourses，则成功得到一个完整的拓扑序列，也就是所有课程都可以修完，如果count<numCourse，则存在环，部分课程无法参加。

Course Schedule

There are a total of n courses you have to take, labeled from 0 to n - 1.

Some courses may have prerequisites, for example to take course 0 you have to first take course 1, which is expressed as a pair: [0,1]

Given the total number of courses and a list of prerequisite pairs, return the ordering of courses you should take to finish all courses.

There may be multiple correct orders, you just need to return one of them. If it is impossible to finish all courses, return an empty array.

For example:
2, [[1,0]]

There are a total of 2 courses to take. To take course 1 you should have finished course 0. So the correct course order is [0,1]

4, [[1,0],[2,0],[3,1],[3,2]]

There are a total of 4 courses to take. To take course 3 you should have finished both courses 1 and 2. Both courses 1 and 2 should be taken after you finished course 0. So one correct course order is [0,1,2,3]. Another correct ordering is[0,2,1,3].

Method

与问题Course Schedule本质相同。课程的组织可以表示为一个有向图，问题本质是检测一个有向图中是否有环，如果一个图有环，则一定存在冲突，也就是环中的课程一定无法参加。对图进行拓扑排序，如果可以进行完整的拓扑排序(输出排序节点等于所有节点数)，则说明是一个有向无环图，否则说明有环。

这里采用基于队列的拓扑排序：

  1. 遍历所有边，统计所有节点的入度(就是修课前需要完成的课程数)，和指向的节点(用HashMap存储)。
  2. 将所有入度为0的节点加入队列。
  3. 在队列非空的情况下，取出队列的一个节点，将这个节点加入结果集，将所有这个节点指向的节点的入度-1，如果每个被指向的节点入度变为0，则将其加入队列。
  4. 循环第3步直到队列为空，如果结果集size==numCourses，则成功得到一个完整的拓扑序列，也就是所有课程都可以修完并且顺序记录在结果集中，返回结果集。如果结果集size<numCourse，则存在环，部分课程无法参加，返回空数组。
