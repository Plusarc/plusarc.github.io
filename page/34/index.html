Implement Stack using Queues

Implement the following operations of a stack using queues.

push(x) – Push element x onto stack.
pop() – Removes the element on top of the stack.
top() – Get the top element.
empty() – Return whether the stack is empty.

Method

由于queue的性质是先进先出，所有要实现stack的pop操作，必须将queue的前n-1个元素pop出，然后才能得到stack需要pop出的元素。对于queue的pop出的前n个元素，直接将重新push到queue中，这样结构上仍然保持了原来push顺序，且不用开辟新空间。

对于top操作，将整个queue的n个元素pop并push回去，并记录最后一个元素值，作为栈顶元素。

Rectangle Area

Find the total area covered by two rectilinear rectangles in a 2D plane.

Each rectangle is defined by its bottom left corner and top right corner as shown in the figure.

Method

求两个矩形在平面上占有的面积，关键是抽象出判断两个矩阵是否有重合，如何求出重合的面积，需要仔细观察。

想象随意给两个矩形， 它们有重合的条件是：

  1. 两个矩形下边的较大值必须 小于 两个矩形上边的较小值
  2. 两个矩形左边的较大值必须 小于 两个矩形右边的较小值

Set Matrix Zeroes

Given a m x n matrix, if an element is 0, set its entire row and column to 0. Do it in place.

Method

扫描matrix，将为0的元素的x，y坐标存入rowSet，colSet中，避免部分重复置0操作。如果colSet或rowSet满了，代表整个矩阵需要置0.清空另一个set，跳出扫描操作。

根据rowSet，colSet存储的行，列置0.

更进一步的优化是，设置rowNotZero作为行元素是否全为0的boolean，和colNotZero的boolean数组用于指示每一列元素是否全为0. 如果为false，代表这些行，列的元素已经全为0，无需重置，根据这些指示值相应的移除set中相应的行，列。这里没有给出实现，只需要加一些判断操作。

Unique Paths II

Follow up for “Unique Paths”:

Now consider if some obstacles are added to the grids. How many unique paths would there be?

An obstacle and empty space is marked as 1 and 0 respectively in the grid.

For example,
There is one obstacle in the middle of a 3x3 grid as illustrated below.

The total number of unique paths is 2.

Note: m and n will be at most 100.

Method

解题思路类似与Unique Paths.

首先要认识到这个问题是可以拆分为子问题的，目标是求得从(1,1)到(m,n)的路径，并每次只能向下或向右移动。假设p(x,y)是从(1,1)到(x,y)的路径数，则p(m,n) = p(m-1,n) + p(m,n-1), 因为到达(m,n)前一个点必为(m-1,n)或(m,n-1)。

这里有个变化是增加了障碍物，在障碍物出把path数设为0就好，其他部分都是一样的处理方式。

Climbing Stairs

You are climbing a stair case. It takes n steps to reach to the top.

Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

Method

动态规划法，对于n>2，其上一步必定是n-1, 或n-2。由n-1跳一步，或者n-2跳两步，所以状态转移方程为：
path(n)=path(n-1)+path(n-2)

递归solution会包含很多子问题的重复计算，建立数组从最下面的台阶开始存储每每个台阶的path，递推方式计算。
