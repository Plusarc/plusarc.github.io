Best Time to Buy and Sell Stock

Say you have an array for which the ith element is the price of a given stock on day i.

If you were only permitted to complete at most one transaction (ie, buy one and sell one share of the stock), design an algorithm to find the maximum profit.

Method

Method 1

对于prices[0],prices[1]...每个值求查找其之后最大值，求得这天购入股票所能获得的最大利润，然后比较所有天购入所分别能获得的最大利润，取这些天的最大值。

会超时。

Method 2

动态规划。其实问题就是获得某天之后(包括当天)的股票价格最大值，可以倒过来计算，从最后一天开始计算。设某天i之后（包括当天）的股票最大值为max[i]，则max[i]= Math.max(price[i], max[i+1])。然后计算当天购入能获得的最大profit为max[i]-price[i]，如果大于当前记录的max_profit，则更新max_profit。

Best Time to Buy and Sell Stock II

Say you have an array for which the ith element is the price of a given stock on day i.

Design an algorithm to find the maximum profit. You may complete as many transactions as you like (ie, buy one and sell one share of the stock multiple times). However, you may not engage in multiple transactions at the same time (ie, you must sell the stock before you buy again).

Method

虽然题目跟Best Time to Buy and Sell Stock类似，但使用的策略是不同的。

首先来看一张股市变化的图：

给定了所有时间的股票价格，最佳策略是什么？ 当然是在局部最低处买入，在同一个对于的下一个局部最高处卖出，其实就是上升子序列，对于所有的上升子序列进行同样的操作。本质上是贪心策略，记入每个上升子序列的非负差值。

所以问题就转化为了检查局部最低处和局部最高处，在局部最低处买入(设置min)，局部最高处卖出(max, profit += (max-min))

Missing Number

Given an array containing n distinct numbers taken from 0, 1, 2, …, n, find the one that is missing from the array.

For example,
Given nums = [0, 1, 3] return 2.

Note:
Your algorithm should run in linear runtime complexity. Could you implement it using only constant extra space complexity?

Method

Method 1

使用Hashset记录。

Method 2

首先数值序列为0,1,2,3,...n, 其总和为n*(n+1)/2.
对数组进行求和得到sum，则缺的那个数字为 n*(n+1)/2 - sum
