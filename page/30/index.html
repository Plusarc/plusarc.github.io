Bitwise AND of Numbers Range

Given a range [m, n] where 0 <= m <= n <= 2147483647, return the bitwise AND of all numbers in this range, inclusive.

For example, given the range [5, 7], you should return 4.

Method

最直接的方法就是对于各个数都做一遍AND操作，明显会TLE。

关键是观察出连续数字的特征。对于连续的两个数n, n+1，其最后一位必定不一样，一个是偶数以0结尾，一个是奇数以1结尾，对于差距为2的两个数n, n+2，其倒数第二位必定不同，一个是0，一个是1，对于差距为4的两个数n, n+4，其倒数第三位必定不同，一个是0，一个是1, …

差距为4的区间n, n+1, n+2, n+3, n+4，所有数进行AND操作其低三位必定都为0，因为n & n+1 =>最低位为0, n & n+2 =>倒数第二位为0 ， n & n+4 =>倒数第三位为0，其他高位均不变，因为n, n+1, n+2, n+3, n+4除低三位的高位都是相同的。

也就是对于一个区间m, n的所有数进行AND操作，其结果就是所有数相同的高位保持原来不变，而之后的低位全部会变为0. 而所有数相同的高位部分可以由比较n和m得到，也就是只需要获得n和m相同的高位部分并保留，对于剩余的低位部分全部置0.

Kth Largest Element in an Array

Find the kth largest element in an unsorted array. Note that it is the kth largest element in the sorted order, not the kth distinct element.

For example,
Given [3,2,1,5,6,4] and k = 2, return 5.

Note:
You may assume k is always valid, 1 ≤ k ≤ array’s length.

Method

很容易想到的一种方法是quick sort之后选择第K大的元素，但是由于我们只需要第K大的元素，没有必要对整个数组进行排序，使用quick sort中的partition方法，可以得到pivot所应该在的位置，左边为比pivot大的元素，右边为比pivot小的元素(根据从大到小的quick sort，从小到大则反过来)。如果pivot的位置刚好为K-1，则说明pivot正好是第K大的元素。如果pivot的位置大于K-1，则第K大的元素在pivot位置的左边，继续搜索左边的数组，如果pivot的位置小余K-1，则第K大的元素在pivot位置的右边，继续搜索右边的数组。

参考quick select.

Ugly Number

Write a program to check whether a given number is an ugly number.

Ugly numbers are positive numbers whose prime factors only include 2, 3, 5. For example, 6, 8 are ugly while 14 is not ugly since it includes another prime factor 7.

Note that 1 is typically treated as an ugly number.

Method

如果一个数是ugly number，通过除以一定次数的2, 3, 5，会正好得到结果1(整除得到的，余数为0)。

所以依次在除2余数为0时，将其不断除以2，直到余数不为0。检查结果是否为1，如果是则返回true。如果不是，则在除3余数为0时，将其不断除以3，直到余数不为0。检查结果是否为1，如果是则返回true。如果不是，则在除5余数为0时，将其不断除以5，直到余数不为0。检查结果是否为1，如果是则返回true,如果不是则说明2，3，5无法乘得该数，也就是该数有其他的prime，不是ugly number，返回false。

Count Primes

Description:

Count the number of prime numbers less than a non-negative number, n.

Method

厄拉多塞筛法。一个数的倍数肯定不是prime number，从2开始，把每个数的倍数全部设为non prime，当达到某一个数m时，如果该数还没有被设置为non prime，则其一定是prime，因为一个non prime的数，假设为m,必定为2-m的求根值的倍数(根据prime的性质)。

建立一个长度为n的boolean数组，从2...n开始，如果其是prime，则设置其所有的倍数数为non-prime，如果是non-prime，则直接跳过(因为non-prime的倍数也必定是non-prime，且已经被设置过了(倍数关系)，避免重复计算)。
