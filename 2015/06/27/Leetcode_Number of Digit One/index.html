Number of Digit One

Given an integer n, count the total number of digit 1 appearing in all non-negative integers less than or equal to n.

For example:
Given n = 13,
Return 6, because digit 1 occurred in the following numbers: 1, 10, 11, 12, 13.

Method

Naive method

对于1..n的每一个数判断包含1的个数，求和。

Method Noname

表达欠佳，建议直接参考下一个方法。

把求1的个数的问题考虑为求每个位出现1的次数，然后求和。如211，个位数1出现的次数为22，10位数1出现的次数为22，百位数1出现的次数为100.那么如何计算这各个位1出现的个数？观察一下211：

个位数1出现的次数为(211/10)*1 + 0-1范围1在个位数出现的次数，也就是1.
十位数1出现的次数为(211/100)*10 + 200-211范围1在十位数出现的次数，也就是210-211
百位数1出现的次数为(211/1000)*100 + 0-211范围内1在百位数出现的次数，也就是100-199

如果是256：

个位数1出现的次数为(256/10)*1 + 0-6范围1在个位数出现的次数，也就是1.
十位数1出现的次数为(256/100)*10 + 200-256范围1在十位数出现的次数，也就是210-219
百位数1出现的次数为(256/1000)*100 + 0-256范围内1在百位数出现的次数，也就是100-199

对于每个位数剩余部分次数取决于其自己与其低位数组成的数与base和2*base-1范围的关系，如211中十位数的计算，11与10, 19的关系，10=<11<19出现1的范围,所以次数为11-10+1。如果是256十位数的计算，则是56与10，19的关系，56>=19,所以次数为base=10，指的是10-19。如果是207十位数的计算，则是7与10，19的关系，7<10,所以次数为0.

Other method

另一种分析方式和解决方法，本质上是一样的,不过选择了从高位到低位处理，比较巧妙清晰。

For example ‘8192’:

1-999 -> countDigitOne(999)
1000-1999 -> 1000 of 1s + countDigitOne(999)
2000-2999 -> countDigitOne(999)
…
7000-7999 -> countDigitOne(999)
8000-8192 -> countDigitOne(192)

Count of 1s : countDigitOne(999)*8 + 1000 + countDigitOne(192)

Noticed that, if the target is ‘1192’:
Count of 1s : countDigitOne(999)*1 + (1192 - 1000 + 1) + countDigitOne(192)
(1192 - 1000 + 1) is the 1s in thousands from 1000 to 1192.
